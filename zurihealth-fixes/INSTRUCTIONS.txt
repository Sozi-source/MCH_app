# ZuriHealth — Play Store Fix Instructions
# Apply all fixes in this order.

=============================================================
FIX 1: API KEY SECURITY (CRITICAL — do this first)
=============================================================

## 1a. Deploy Supabase Edge Functions

From your project root in PowerShell:

  npx supabase login
  npx supabase link --project-ref 26fd1a97-b4cb-421b-99d7-24428a1d0fe6

Copy the three Edge Function folders into your project:

  supabase/
    functions/
      zuri-chat/index.ts      ← AI chat
      zuri-zscore/index.ts    ← Z-score calculation
      zuri-meal/index.ts      ← Meal suggestions

Then deploy all three:

  npx supabase functions deploy zuri-chat
  npx supabase functions deploy zuri-zscore
  npx supabase functions deploy zuri-meal

## 1b. Set the Groq API key as a secret (NOT in .env)

  npx supabase secrets set GROQ_API_KEY=your_actual_groq_key_here

## 1c. Update your .env file

Remove these two lines from .env:
  EXPO_PUBLIC_GROQ_API_KEY=...
  EXPO_PUBLIC_ANTHROPIC_API_KEY=...   (if present)

Keep ONLY these two:
  EXPO_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
  EXPO_PUBLIC_SUPABASE_ANON_KEY=your-anon-key

## 1d. Replace src/lib/zscore.ts

Replace the entire file with the one provided in:
  fixes/src/lib/zscore.ts

## 1e. Patch src/app/(tabs)/chat.tsx

Open chat.tsx and make these changes:

  REMOVE from top of file:
    const GROQ_API_URL = ...
    const GROQ_API_KEY = ...
    const GROQ_MODEL   = ...

  ADD this import (if supabase not already imported):
    import { supabase } from '@/lib/supabase';

  REPLACE the sendMessage function with the one in:
    fixes/src/app/tabs/chat.PATCH.ts

## 3a. Create the adaptive icon

You need TWO separate icon files:

  assets/images/icon.png
    → 1024×1024px, no transparency, used for the Play Store listing icon.
    → This already exists. Ensure it has NO transparent background.

  assets/images/adaptive-icon.png
    → 1024×1024px WITH transparency
    → The ZuriHealth logo/mark centered with generous padding (~20% on each side)
    → The logo should NOT fill the entire canvas — leave room for Android to
       apply its own shape masking (circle, squircle, etc.)
    → Background will be supplied by app.json backgroundColor: "#E6F4FE"

To create this quickly in PowerShell using ImageMagick (if installed):
  magick assets/images/icon.png -resize 614x614 -gravity center `
    -background none -extent 1024x1024 assets/images/adaptive-icon.png

Or open your icon in any image editor, paste it centered on a transparent
1024×1024 canvas with padding, and export as PNG.

## 4b. Host a public privacy policy URL

The Play Store requires a publicly accessible URL (not just in-app).

Option A — GitHub Pages (free, 5 minutes):
  1. Create a file docs/privacy.html in your repo
  2. Enable GitHub Pages on the repo (Settings → Pages → branch: main, folder: /docs)
  3. Your URL will be: https://sozi-source.github.io/MCH_app/privacy.html

Option B — Vercel (free):
  Create a simple HTML file and deploy it.

The privacy policy content from privacy.tsx is your source — just convert it
to a simple HTML page. Submit this URL in Play Console → App content → Privacy policy.
